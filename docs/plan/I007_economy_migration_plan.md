# I-007 경제 이관 — 구현 작업 플래닝 (게임 레포)

> **근거:** `spec:docs/ideas/I-007_SkillTreeManastoneCharmEconomy.md` (§11 프로토 순서·§15 Phase 로드맵). 스펙은 P0~P5 **로컬 SSOT 패치 완료**, 게임 **구현은 대부분 미진행**. 본 문서 = 미진행분의 **구현 순서·마일스톤·DoD**.
> **작성:** 2026-07-09. 결속(I-008)·템포(I-006)는 게임 구현 완료(별건). 본 플랜은 **경제 대개편(I-007)** 한정.

## ⚠️ 대전제 — 이건 "추가"가 아니라 "교체"다

게임은 지금 I-007이 **폐기 대상**으로 지목한 구 시스템으로 돌아간다:
- **구 F-009 스킬북 경제** — 분석 N=3→해금→상점(`hub_economy_panel`, P2-S6b), 스킬북 인스턴스·charges·per-kill 드롭.
- **gear affix** — gear 롤테이블 G1~G3(dmg/cd/potency band).

I-007 구현 = 위를 **뜯어내고** 마석·참·스킬트리·금고재료로 대체 + 세이브 마이그레이션. 파괴적이라 **경제 스캐폴딩을 먼저 세운 뒤 F-009를 폐기**해야 함(§순서 주의). 순서 뒤집으면 "그라인드만 남는" 공백 발생(I-007 §15 피해야 할 순서).

## 구현 마일스톤 (I-007 §11 프로토 순서 기반)

경제 스캐폴딩(M1~M4) → 폐기·교체(M5) → 잭팟 레이어(M6). 각 마일스톤 = 플테 가능한 최소 덩어리.

### M1 — 마석 (manastone) *(스캐폴딩 시작점)*
- 슬롯 스킬(Q/E/R) 발동 시 **마석 소모**. 스킬 정의는 coeff·행동(기존 유지), 위력 발동=마석.
- 드롭: 일반 적→약한 마석 / 3세력→강한 마석. 시그니처=무소모/극저(NC 3인 고갈 방지).
- 정산: 런 At Risk → 탈출 → stash(기존 `F-007` 골격 재사용). 고갈=의도적 고역(소프트락 아님, UI 시전 거부).
- **과도기 트릭:** 초기엔 스킬북 charges를 마석 카운터로 재해석 가능(I-007 P2-1 주). 완전 분리는 M5.
- **DoD:** 슬롯 스킬이 마석을 소모하고, 마석이 드롭·stash되며, 고갈 시 시전 거부 + 경고.

### M2 — 참 (charm)
- 인벤 그리드(`F-010` 백팩) 점유 오오라. 3~5종(원소 부여/저항/증폭/반사 등 — **액티브 아님**).
- 들고 있을 때만 적용(칸 vs 파워 긴장). At Risk → 탈출 → stash.
- **연동:** §3.1 "stat·오오라=참" — 현재 gear roll이 담당하던 stat 축을 참으로 이전 검토(affix 폐기와 함께).
- **DoD:** 참 3~5종이 인벤 점유·오오라 적용·정산되고, stat이 참으로 표현됨.

### M3 — 스킬 트리 (skill tree)
- 허브 트리: **노드 해금**(AB 사용 허용) + **업그레이드 노드**(하위 행동 발전, `ROLE-001` OPENER→PAYOFF). stat 증폭은 트리 아님(참).
- 트리 단위(클래스/gear archetype/Identity별 분기)는 §14.7 미결 — 구현 시 확정.
- **연동:** 구 F-009 분석 N=3→해금을 트리 해금으로 **교체**.
- **DoD:** 허브에서 트리로 스킬 해금·업그레이드, 던전 진입 전 빌드 계획 성립.

### M4 — 금고 통합 + gear 슬롯 소멸 모델
- Shared 적 AB 처치 → **재구현 재료** 드롭(즉시 스킬북 장착 폐지) → 허브 **금고**(D-029 haul과 동일 축·탭 통합).
- 재료 = 트리·Shared 해금·시설 승급 **소비**(허브 sink 경쟁, 구 분석 딜레마 대체).
- gear 교체 시 끼운 슬롯 스킬 **소멸** → 상점 재구매·신 gear 재모딩(건 모딩 UI). 이중용도 없음(금고 적재=Safe).
- **DoD:** 처치→재료→금고 적재→해금/승급 소비 루프 + gear 교체 시 슬롯 소멸·재구매.

### M5 — F-009 폐기 + affix Deprecated *(파괴적 교체 — M1~M4 선행 필수)*
- `hub_economy_panel` 분석·상점·스킬북 인스턴스·charges·per-kill 드롭 **제거**.
- gear 롤테이블 affix(dmg/cd/potency) **폐기 또는 극소화** — 빌드 변주는 `rolledIdentitySkillId`+결속으로.
- **세이브 마이그레이션:** 기존 스킬북/affix 인스턴스 → 마석·참·재료로 변환 또는 만료 처리.
- **DoD:** 구 경제 코드·데이터 제거, 세이브 마이그레이션 통과, ci_smoke·hub_smoke green.

### M6 — 잭팟 레이어 (마석 티어·던전 전용·공명 sink)
- 상점: 해금 단계별 마석 티어 확대. 최상급 마석=**특정 던전 루팅 전용**(상점 미판매).
- 3세력: 강한 마석 + 저확률 Identity gear. 결속 **공명 충전**(금고 재료 sink) — BiS 고착 방어(§14.9).
- **DoD:** 마석 티어·던전 전용·3세력 gear·공명 sink가 잭팟/sink로 작동.

## 순서·의존

```
M1 마석 ─┐
M2 참   ─┼─ (경제 스캐폴딩, 병렬 가능) ─→ M4 금고+소멸모델 ─→ M5 F-009·affix 폐기 ─→ M6 잭팟
M3 트리 ─┘                                    ↑ (트리 해금이 F-009 분석 대체라 M5 전 필요)
```
- **선행:** 각 마일스톤은 플테로 검증 후 다음. M5(폐기)는 M1~M4가 대체재를 다 세운 뒤에만.
- **교차:** 참(M2)이 stat을 흡수해야 affix 폐기(M5)가 공백 없음. 트리(M3)가 분석 해금을 대체해야 M5 가능.

## 미결 (I-007 §14.7 — 구현 시 확정)
- 참 크기·스택 상한·stash 용량 관계.
- 스킬 트리 단위(클래스/archetype/Identity 분기).
- 상점 스킬 재구매 가격·티어·트리 선행 조건.
- Identity 무결도/등급 필드명(affix 서사 대체).

## 스펙 정합
- 스펙 SSOT는 P0~P5 로컬 패치 완료(I-007 §15). 본 구현이 스펙과 어긋나면 **DRIFT 로깅** → 필요 시 OPS_30.
- 승격 문서(I-007 §13): `F-009`(대체)·`F-020`·`F-008`·`F-007`·`F-029`·`D-016`·UI. 구현 착수 시 각 참조.
